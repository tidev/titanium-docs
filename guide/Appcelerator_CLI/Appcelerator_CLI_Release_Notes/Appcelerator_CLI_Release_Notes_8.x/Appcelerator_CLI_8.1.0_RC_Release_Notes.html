# Appcelerator CLI 8.1.0 RC - 14 August 2020

Appcelerator CLI 8.1.0 RC is a release candidate for a minor release that includes new features, improvements, and bug fixes.

# Component versions

The following components are shipped with CLI 8.1.0:

Alloy

1.15.1

Cloud CLI

2.1.3

Titanium CLI

5.2.2

Daemon

3.2.0

# New Features

  • ALOY-1732 (opens new window) - Add ability to run in the background without UI

    • This change is dependent on the to be released Titanium SDK ticket TIMOB-27896 (opens new window) and is shipping ahead of time to make upgrades to that version smoother

# Improvements

  • ALOY-1253 (opens new window) - Alloy: Support WPATH in widget XML view attribute values

  • ALOY-1733 (opens new window) - Add templates for webpack usage

  • ALOY-1735 (opens new window) - Add vscode settings file to projects

  • CLI-1391 (opens new window) - Allow specifying Native Module templates

    • Module templates will now be prompted for when creating a module. You can also specify them by using the --android-code-base and the --ios-code-base options

  • CLI-1392 (opens new window) - Improve handling of no MBaaS entitlement

  • CLI-1393 (opens new window) - Allow passing template option through when creating a classic application

    • The --template option will now be respected when creating an application with the --classic flag

# Fixed issues

  • ALOY-1256 (opens new window) - Widget/Require as top-level element uses controller as view

  • ALOY-1720 (opens new window) - Version string comparison will break for SDK 10.0.0

  • ALOY-1734 (opens new window) - JS files imported in "alloy.js" don't have access to Alloy globals as of Titanium 9.0.0

  • ALOY-1736 (opens new window) - aloy new fails if dev dependencies do not exist in package.Json
